# FlowTree 桌面网页原型

这是 FlowTree 任务树洞的桌面端单页应用。它支持本机模式，也可以接入 Supabase，启用账户登录、云端保存、跨设备同步和用户数据隔离。

## 运行

直接用浏览器打开 `index.html`，或在本目录启动静态服务：

```bash
python3 -m http.server 8799
```

访问：

```text
http://127.0.0.1:8799/
```

## 页面

顶部导航包含：

1. 首页：脑内输入、语音入口、最近 5 条输入、草稿保存。
2. 正在做：集中显示每个看板中被插旗的当前任务。
3. 今日任务：仅保留“今天要推进”和“已完成”。
4. 总任务看板：长期项目列表，今日任务更多的看板优先。
5. 记事本：汇总想法、情绪、资料、决策和复盘，支持标签总览与搜索。

点击总任务看板或今日任务后，会进入独立的具体任务看板。

AI 待确认不是一级页面。它只在首页执行 AI 整理后作为临时确认流程出现，确认后的任务进入看板，其他内容进入记事本。

## 导航与用户动线

顶部导航只表达五个稳定的一级分区：

1. 首页：捕捉脑内输入，并发起 AI 整理。
2. 正在做：查看所有插旗任务并进入具体任务。
3. 今日任务：执行今天要推进的任务。
4. 总任务看板：选择或创建长期任务空间。
5. 记事本：查看和管理全部记录。

以下内容属于流程或详情状态，不占用一级导航，也不会让某个一级导航保持选中：

- AI 整理确认：`首页 → AI 整理 → 确认本次整理 → 任务看板 / 记事本`。
- 具体任务看板：`总任务看板或今日任务 → 具体任务看板 → 当前任务`。

具体任务看板使用页面内面包屑表达位置：

```text
返回来源 / 总任务看板名称 / 父任务 / 当前任务
```

返回来源由实际入口决定。从今日任务进入就返回今日任务，从总任务看板进入就返回总任务看板；通过其他一级页面的运行中番茄钟进入时，也返回原页面。刷新或直接打开具体任务地址时，默认返回总任务看板。

顶部左箭头在所有页面提供“返回上一个界面”。具体任务页中，项目和父任务面包屑也可以点击跳转。

## 具体任务看板

- 左侧是可折叠任务树导航。
- 中间是面积最大的当前任务画板。
- 支持无限层级任务，前三层使用不同视觉权重。
- 每个看板只能插旗一个当前任务，插旗不改变任务位置。
- 已完成任务保留在原位置并默认折叠。
- 选中任意层级任务后，中间显示该任务专属记事区，可保存文字、图片和标签。
- 任务记录同时出现在全局记事本，并保留对应任务入口。
- 单条任务记录最多添加 4 张图片；图片会压缩后存入当前浏览器的 `localStorage`，仍不支持跨设备同步。
- “任务设置”可配置计划日期/时间、截止日期/时间、价值和紧急程度。
- 计划日期设为今天时自动加入今日任务；“加入今日”也会把计划日期设为今天。
- 截止日期仅用于提示，不会自动加入今日任务。
- “复盘一下”只在任务完成后出现。
- 活动记录通过右侧临时抽屉查看。
- 番茄钟支持自定义专注/休息时长、暂停、重置和刷新恢复。

## 记事本

所有页面右下角都有快速记录入口，完整内容可在“记事本”查看。

- 支持“全部备注”和“按标签”视图。
- 正文内可直接输入 `#标签`。
- 标签可自动识别、补充和删除。
- 支持搜索、编辑和删除。
- AI 提取出的想法、情绪、资料和决策确认后进入记事本。

## 数据存储与迁移

未配置 Supabase 时，数据保存在浏览器 `localStorage`：

```text
flowtree_state_v2
```

首次打开新版时：

- 如果存在 `flowtree_state_v1`，会自动迁移项目、任务、历史和待确认内容。
- 旧每日看板的未完成栏目合并为“今天要推进”。
- 旧“已完成”栏目迁移到新版“已完成”。
- 旧项目根任务会转换为看板容器，子任务成为新的一级任务。
- 如果没有旧数据，则载入示例数据。

配置 Supabase 后：

- 用户使用邮箱和密码注册、登录。
- 每个账户的数据保存到 `flowtree_user_state`，并由 Row Level Security 隔离。
- 每个账户使用独立的本地缓存，断网时仍可读取最近一次数据。
- 恢复网络或重新聚焦页面时，会检查云端的新版本。
- 旧 `flowtree_state_v2` 数据只会在第一个账户首次登录时迁移一次。
- 任务记录图片保存在私有 Storage bucket，不再写入云端 JSON。

### 启用云端

1. 创建 Supabase 项目。
2. 在 Supabase SQL Editor 执行 `supabase-schema.sql`。
3. 在项目设置中复制 Project URL 和前端可用的 publishable/anon key。
4. 填入 `cloud-config.js`：

```js
window.FLOWTREE_CONFIG = {
  supabaseUrl: "https://YOUR_PROJECT.supabase.co",
  supabaseAnonKey: "YOUR_PUBLISHABLE_OR_ANON_KEY",
  stateTable: "flowtree_user_state",
  imageBucket: "flowtree-task-images",
};
```

5. 在 Supabase Authentication 的 URL Configuration 中，把正式网站地址加入 Site URL 和 Redirect URLs。
6. 将本目录部署到任意静态网站平台。

`service_role` key 拥有绕过 RLS 的权限，禁止写入 `cloud-config.js` 或任何浏览器代码。

## 本轮暂不实现

以下能力已进入后续计划，当前数据结构和界面已为其预留扩展位置：

1. 真实 AI 任务提取 API 与语音转写 API。
2. 任务树拖拽排序。
3. 拖拽改变父子关系和整分支移动。
4. 手机网页适配。
5. SwiftUI iOS App。

真实 AI 接入时不要把 API Key 写入前端，应通过后端代理调用。

## 回归检查

- v1 到 v2 数据迁移。
- 五个一级页面和具体任务看板渲染。
- AI 任务/其他分区。
- 今日两栏。
- 无限层级任务和分支完成。
- 记事本标签提取。
